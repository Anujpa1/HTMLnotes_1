Que1. Difference between div and span?

The main difference between Div and Span is their default display behavior and the type of content they are typically used for:

DIV: is a block-level element that is used to group and separate content into distinct sections or containers. By default, elements start on a new line and take up the full width available to them. They are commonly used for larger structural elements of a web page, such as header, footer, sidebar, or main content areas. You can apply CSS styles and manipulate the layout of elements using properties like width, height, margin, padding, etc.

SPAN: is an inline-level element used to apply styles or target specific portions of text within a larger block of content. Unlike div, span elements do not start on a new line by default. They are typically used to add inline styling, apply CSS classes or IDs, or provide hooks for JavaScript interactions. span is often used within paragraphs or headings to style specific words or phrases differently.

Que4. How many types of headings are there ?How to decide which to use where

HEADINGS OF HTML: In HTML, there are six levels of headings, designated by the h1 to h6 tags. Each heading level represents a different level of importance and hierarchy in the structure of the content. The heading levels are as follows: h1: This represents the highest level of heading and is typically used for the main heading of a page or a major section. h2: This represents the second level of heading and is used for subheadings or sections within the main content. h3: This represents the third level of heading and is used for sub-sections within the h2 headings. h4, h5, h6: These represent lower levels of headings and are used for sub-sections within higher-level headings or for more granular organization of content. When deciding which heading level to use, it is important to consider the structure and hierarchy of your content. The heading levels should reflect the logical organization of the information on your page. Here are some general guidelines: Use h1 for the main title of the page. There should typically be only one h1 per page, representing the main focus or purpose of the content. Use lower-level headings (h2, h3, etc.) for subheadings or sections within the content. Each section should have a clear relationship to the higher-level heading it falls under. Follow a consistent hierarchical structure. For example, if you have an h2 heading, the sections within it should use h3 headings, and any subsections within those should use h4 headings, and so on. Avoid skipping heading levels. It is generally recommended to follow a sequential order and not jump from, for example, h2 to h4, as it can disrupt the logical flow and hierarchy of the content. By using appropriate heading levels, you can enhance the accessibility, readability, and organization of your content. Headings also play a role in search engine optimization (SEO), as search engines use them to understand the structure and importance of the content on your page.

Block-level elements: Block-level elements create a block-level box that occupies the full width available to them and starts on a new line. Example: DIV, P, H1 TO H6, UL, LI, SECTION, ARTICLE

Que9. What are semantic tags?

1. header: Represents the introductory or navigational section of a document or a section within it.
2. nav: Defines a section of navigation links.
3. main: Represents the main content of a document.
4. article: Defines an independent, self-contained piece of content, such as a blog post or news article.
5. aside: Defines content that is tangentially related to the main content, often presented in a sidebar or as a separate block.
6. section: Represents a thematic grouping of content within a document.
7. footer: epresents the footer section of a document or a section within it, typically containing information like copyright notices, contact details, or sitemap links.
8. h1 to h6: Heading tags that represent different levels of section headings, with h1 being the highest and h6 being the lowest.

Que10. Difference between HTML tags and elements?

1. HTML Tags: HTML tags are the building blocks of an HTML document. They are used to mark up and define the structure and elements within the document. HTML tags are written as pairs with an opening tag and a closing tag, and they enclose the content that they affect. For example, p is an opening tag for a paragraph, and /p pis the corresponding closing tag. Tags define the beginning and end of an element.

2. HTML ELEMENTS: HTML elements consist of an opening tag, content, and a closing tag. They are composed of one or more HTML tags and the content placed between those tags. The content can be text, images, links, or other elements. For example, p is the opening tag of a paragraph element, the content is the text or other elements within the paragraph, and /p is the closing tag.
